PHẦN 1 — EXPLORATORY DATA ANALYSIS (EDA)

EDA là bước đầu tiên và quan trọng nhất trong bất kỳ dự án data science nào. Mục tiêu không phải là "vẽ chart cho đẹp", mà là hiểu dữ liệu đang nói lên điều gì trước khi xây model hay đưa ra kết luận.

📌 Tại sao EDA quan trọng?
80% thời gian của data scientist là dành cho data understanding và cleaning. EDA giúp phát hiện outlier, missing values, distribution lệch, và correlation ẩn — những yếu tố quyết định model sẽ hoạt động tốt hay không.

1.3 Phân phối dữ liệu & Outlier Detection

Nhận diện Distribution Shape

Trước khi làm gì với data, phải biết nó có hình dạng gì:

ShapeĐặc điểmXử lý
NormalMean ≈ Median, bell curveDùng trực tiếp, StandardScaler
Right-skewedĐuôi dài về phải, mean > medianLog transform: np.log1p(x)

Outlier Detection — 3 phương pháp

# Phương pháp 1: IQR (Interquartile Range) — phổ biến nhất
Q1 = df['revenue'].quantile(0.25)
Q3 = df['revenue'].quantile(0.75)
IQR = Q3 - Q1
lower = Q1 - 1.5 * IQR
upper = Q3 + 1.5 * IQR
outliers = df[(df['revenue'] < lower) | (df['revenue'] > upper)]
print(f'Outliers: {len(outliers)} rows ({len(outliers)/len(df)*100:.1f}%)')

# Phương pháp 2: Z-score — tốt cho normal distribution
from scipy import stats
z_scores = np.abs(stats.zscore(df['revenue']))
outliers_z = df[z_scores > 3]  # >3 std devs from mean

# Phương pháp 3: Isolation Forest — cho multivariate outlier
from sklearn.ensemble import IsolationForest
clf = IsolationForest(contamination=0.05, random_state=42)
df['is_outlier'] = clf.fit_predict(df[['revenue', 'cogs', 'quantity']])
# -1 = outlier, 1 = normal
⚠️ Quan trọng
Khi phát hiện outlier, đừng xóa ngay! Hỏi: "đây là lỗi data hay insight thực sự?" Ví dụ: ngày doanh thu tăng đột biến có thể là Black Friday, không phải lỗi.

                              corr_pearson.columns[j],

2.1 KPI Design — Thiết kế chỉ số kinh doanh

Các KPI tài chính cốt lõi

KPICông thứcÝ nghĩa
RevenueΣ (quantity × unit_price)Tổng doanh thu thuần
Gross ProfitRevenue − COGSLợi nhuận gộp trước chi phí vận hành
Gross Margin %(Revenue−COGS) / Revenue × 100Tỷ suất lợi nhuận gộp (%)
CAGR(End/Start)^(1/n) − 1Tăng trưởng kép hàng năm qua n năm
AOVRevenue / Số đơn hàngGiá trị đơn hàng trung bình
Churn RateKH mất / Tổng KH đầu kỳ × 100% KH ngừng mua trong một kỳ
LTVAOV × Frequency × LifespanGiá trị KH mang lại suốt vòng đời

Code tính KPI

import pandas as pd, numpy as np

# Revenue & Gross Profit
df['line_revenue']     = df['quantity'] * df['unit_price']
df['line_cost']        = df['quantity'] * df['cogs']
df['line_gross_profit']= df['line_revenue'] - df['line_cost']
df['gross_margin_pct'] = (df['line_gross_profit'] / df['line_revenue'] * 100).round(2)

# CAGR (2013 → 2022 = 9 năm)
rev_2013 = df[df['year']==2013]['revenue'].sum()
rev_2022 = df[df['year']==2022]['revenue'].sum()
cagr = ((rev_2022 / rev_2013) ** (1/9) - 1) * 100

# AOV (Average Order Value)
aov = df.groupby('order_id')['payment_value'].sum().mean()

# MoM Growth
monthly = df.groupby('year_month')['revenue'].sum().reset_index()
monthly['prev_revenue'] = monthly['revenue'].shift(1)
monthly['mom_growth'] = (monthly['revenue'] - monthly['prev_revenue']) / monthly['prev_revenue'] * 100

# YoY Growth
annual = df.groupby('year')['revenue'].sum().reset_index()
annual['yoy_growth'] = annual['revenue'].pct_change() * 100

2.2 RFM Analysis — Phân khúc khách hàng

RFM là phương pháp phân tích hành vi mua hàng dựa trên 3 chiều: Recency, Frequency, Monetary. Đây là nền tảng của mọi chiến lược CRM hiện đại.

ChiềuĐịnh nghĩaCâu hỏi trả lời
R — RecencySố ngày kể từ lần mua cuốiKH mua gần đây không? R thấp = mua gần = tốt hơn
F — FrequencySố lần mua hàngKH mua thường xuyên không? F cao = trung thành hơn
M — MonetaryTổng giá trị mua hàngKH chi bao nhiêu tiền? M cao = giá trị cao hơn

Code đầy đủ RFM

# Bước 1: Chuẩn bị dữ liệu
ref_date = orders['order_date'].max() + pd.Timedelta(days=1)
active_orders = orders[orders['order_status'] != 'cancelled']

# Bước 2: Tính R, F, M cho từng khách hàng
rfm = active_orders.merge(payments[['order_id','payment_value']], on='order_id')
rfm_agg = rfm.groupby('customer_id').agg(
    last_order = ('order_date', 'max'),
    frequency  = ('order_id', 'nunique'),
    monetary   = ('payment_value', 'sum'),
    avg_order  = ('payment_value', 'mean'),
).reset_index()
rfm_agg['recency_days'] = (ref_date - rfm_agg['last_order']).dt.days

# Bước 3: Gán score (1-5) theo quintile
rfm_agg['R'] = pd.qcut(rfm_agg['recency_days'], q=5, labels=[5,4,3,2,1]).astype(int)
rfm_agg['F'] = pd.qcut(rfm_agg['frequency'].rank(method='first'), q=5, labels=[1,2,3,4,5]).astype(int)
rfm_agg['M'] = pd.qcut(rfm_agg['monetary'].rank(method='first'), q=5, labels=[1,2,3,4,5]).astype(int)
rfm_agg['RFM_score'] = rfm_agg['R']*100 + rfm_agg['F']*10 + rfm_agg['M']

# Bước 4: Gán segment label
def rfm_segment(row):
    r, f, m = row['R'], row['F'], row['M']
    if r >= 4 and f >= 4:                   return 'Champions'
    elif r >= 3 and f >= 3:                 return 'Loyal Customers'
    elif r >= 4 and f <= 2:                 return 'New Customers'
    elif r >= 3 and m >= 3:                 return 'Potential Loyalists'
    elif r <= 2 and f >= 3:                 return 'At Risk'
    elif r <= 2 and f >= 4 and m >= 4:      return "Can't Lose Them"
    elif r <= 2 and f <= 2:                 return 'Lost'
    else:                                   return 'Need Attention'

rfm_agg['segment'] = rfm_agg.apply(rfm_segment, axis=1)

# Bước 5: Phân tích kết quả
summary = rfm_agg.groupby('segment').agg(
    count        = ('customer_id', 'count'),
    avg_recency  = ('recency_days', 'mean'),
    avg_frequency= ('frequency', 'mean'),
    avg_monetary = ('monetary', 'mean'),
).round(1)
summary['pct'] = (summary['count'] / summary['count'].sum() * 100).round(1)
print(summary.sort_values('avg_monetary', ascending=False))

Các segment và chiến lược hành động

SegmentRFM ProfileChiến lược
ChampionsR↑ F↑ M↑VIP program, early access, upsell premium
Loyal CustomersR↑ F↑ M~Reward points, referral program để giữ chân
At RiskR↓ F↑ M↑Win-back email + discount, nhắc nhở cá nhân hóa
New CustomersR↑ F↓ M~Welcome sequence, first-repeat purchase incentive
LostR↓ F↓ M↓Chi phí reactivate cao, chỉ re-engage nếu LTV cao

2.3 Cohort Analysis — Phân tích nhóm theo thời gian

📊 Cohort là gì?
Cohort = nhóm khách hàng có cùng đặc điểm xuất hiện trong cùng khoảng thời gian (thường là tháng mua đầu tiên). Retention Rate = % khách còn mua ở tháng thứ N so với tháng đầu.

Tại sao Cohort Analysis quan trọng? Nó trả lời câu hỏi: "Chương trình cải thiện sản phẩm tháng 6/2022 có giúp giữ chân KH tốt hơn không?" — điều mà aggregate metrics không thể phát hiện.

Code Cohort Retention Matrix

# Bước 1: Xác định tháng mua đầu tiên (cohort) của mỗi KH
first_order = orders[orders['order_status'] != 'cancelled'] \
    .groupby('customer_id')['order_date'].min().reset_index()
first_order.columns = ['customer_id', 'cohort_date']
first_order['cohort_month'] = first_order['cohort_date'].dt.to_period('M')

# Bước 2: Join và tính period number
df_cohort = orders[orders['order_status'] != 'cancelled'].copy()
df_cohort['order_month'] = df_cohort['order_date'].dt.to_period('M')
df_cohort = df_cohort.merge(first_order[['customer_id','cohort_month']], on='customer_id')

# period_number = số tháng kể từ lần mua đầu
df_cohort['period'] = (
    (df_cohort['order_month'].dt.year  - df_cohort['cohort_month'].dt.year) * 12 +
    (df_cohort['order_month'].dt.month - df_cohort['cohort_month'].dt.month)
)

# Bước 3: Tính cohort size và retention
cohort_size = df_cohort.groupby('cohort_month')['customer_id'].nunique()

retention = df_cohort.groupby(['cohort_month','period'])['customer_id'].nunique()
retention = retention.reset_index()
retention.columns = ['cohort_month', 'period', 'customers']

# Pivot thành ma trận
retention_pivot = retention.pivot_table(
    index='cohort_month', columns='period', values='customers'
)
# Chia cho cohort size để ra %
retention_pct = retention_pivot.divide(cohort_size, axis=0) * 100
retention_pct = retention_pct.iloc[:, :24].round(1)  # chỉ lấy 24 tháng

# Bước 4: Visualize
fig, ax = plt.subplots(figsize=(16, 8))
sns.heatmap(retention_pct, annot=True, fmt='.0f', cmap='YlOrRd_r',
            vmin=0, vmax=100, ax=ax, linewidths=0.3,
            cbar_kws={'label': 'Retention Rate (%)'},
            annot_kws={'size': 8})
ax.set_title('Cohort Retention Matrix', fontsize=14, fontweight='bold')
ax.set_xlabel('Months Since First Purchase')
ax.set_ylabel('Cohort (First Purchase Month)')

Đọc kết quả Cohort Matrix

  • Period 0: Luôn = 100% (KH mua lần đầu)
  • Period 1: % KH quay lại tháng ngay sau → đây là số quan trọng nhất
  • Hàng nào sáng hơn: cohort đó có retention tốt hơn trung bình
  • Cột nào tối đột ngột: sự kiện gì xảy ra tháng đó (promo hết, UX thay đổi)?
🎯 Pattern cần chú ý
Nếu retention rơi mạnh từ Period 0 xuống Period 1 (ví dụ từ 100% xuống còn 5-10%) → khách mua một lần rồi biến mất → cần welcome sequence + first-repeat incentive ngay sau lần mua đầu.

2.5 Tổng hợp — Build KPI Dashboard bằng Python

Kết hợp tất cả phần trên thành một script hoàn chỉnh tính toán và xuất report.

def build_business_dashboard(orders, order_items, payments, customers, products):
    """Tính toán toàn bộ KPI và trả về dict kết quả."""
    
    # Revenue KPIs
    fact = order_items.merge(orders[['order_id','order_date','order_status',
                                     'order_source','customer_id']], on='order_id')
    fact = fact.merge(products[['product_id','price','cogs']], on='product_id')
    fact = fact.merge(payments[['order_id','payment_value']], on='order_id')
    
    fact['line_rev']    = fact['quantity'] * fact['unit_price']
    fact['line_cost']   = fact['quantity'] * fact['cogs']
    fact['gross_profit']= fact['line_rev'] - fact['line_cost']
    
    annual = fact.groupby(fact['order_date'].dt.year)['line_rev'].sum()
    cagr   = ((annual.iloc[-1]/annual.iloc[0])**(1/(len(annual)-1))-1)*100
    
    # Customer KPIs
    aov   = payments['payment_value'].mean()
    freq  = orders.groupby('customer_id')['order_id'].nunique()
    repeat_rate = (freq >= 2).mean() * 100
    
    results = {
        'total_revenue'    : fact['line_rev'].sum(),
        'gross_margin_pct' : fact['gross_profit'].sum()/fact['line_rev'].sum()*100,
        'cagr_pct'         : cagr,
        'aov'              : aov,
        'repeat_rate_pct'  : repeat_rate,
        'top_category'     : fact.groupby('category')['line_rev'].sum().idxmax(),
    }
    return results

  • Tìm outlier bằng IQR → quyết định giữ hay xóa
  • Correlation matrix → identify multicollinearity
  • Time series plot → nhận diện trend, seasonality, anomaly
  • Pair plot → overview tất cả relationships
  • Segment analysis → so sánh theo category, region, time
